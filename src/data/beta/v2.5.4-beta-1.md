# Enconvo 2.5.4 Beta 1

- [100%] Cleaner macOS app package
  - Removed unused SPM packages: OpenAISwift, ActivityIndicatorView, SwiftUIPager, OpenAI by MacPaw, Blackbird, GoogleSignIn, GoogleSignInSwift, MenuBarExtraAccess, MSDisplayLink, SpringInterpolation, and Splash.
  - Deleted the unused WindowAnimation helpers and cleaned dead Swift imports in views and services.
  - Kept AudioKit because it is still used by active audio features.

- [100%] More privacy-conscious analytics
  - Identifies signed-in users consistently across the Mac app, Node.js runtime, and web settings app.
  - Filters sensitive values such as API keys, passwords, and tokens before analytics events are sent.
  - Uses clearer event names for command runs, panels, providers, settings, login/logout, and feature flags.

- [100%] Dynamic Island toggle in menu bar
  - Added Enable/Disable Dynamic Island menu item to the status bar menu.

- [100%] Sidebar Ask action
  - New "Ask Sidebar" command that opens the app sidebar with selected text or file context.
  - Registered as a Finder service and general action for right-click integration.

- [90%] Beta update notes
  - Beta update prompts can show beta-specific notes from the active beta changelog.
  - Beta notes stay separate from public production changelogs until the production release.

- [90%] Dynamic Island event handling refactor
  - Moved event handling to DynamicIslandContainerView+HandleEvent extension matching Smart Bar patterns.
  - Uses HotKeyManager stack-based monitors instead of raw NSEvent monitors.
  - Escape and Cmd+W use closePanel(dismissOnly: true); Cmd+N handled in event handler.
  - Fixed paste shortcut race condition caused by competing NSEvent monitors on Return key.
  - Monitor registration tied to panelState.isKeyAndShow for reliable open/close cycles.

- [90%] Disable auto-read text display in Dynamic Island
  - Removed streaming text preview from compact and expanded DI modes when agent auto-read is active.


- [80%] Notch area exclusion for Dynamic Island hover
  - New "Exclude Notch Area" setting (dynamicIsland.excludeNotchArea, default off).
  - When enabled, mouse hover inside the physical notch rect does not trigger DI activation.
  - Hover activation from below and beside the notch still works, allowing coexistence with other notch apps.
