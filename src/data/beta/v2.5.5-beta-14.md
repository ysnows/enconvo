# Enconvo 2.5.5 Beta 14

Thanks for testing Enconvo 2.5.5 Beta 14. This build brings deterministic site
automation to the App Sidebar, rounds out its app presets, and makes chat
attachments and recordings easier to work with.

## Highlights

- **Command Packs bring deterministic site automation to the App Sidebar** —
  installable per-site command sets that search, read rankings, feeds, and
  comments, and check login state through your real browser session.
- **Figma and Palmier Pro join the App Sidebar's app presets**, each pinning
  the right MCP server (and, for Figma, its fourteen official skills) for
  one-click automation.
- **Chat attachments now preview inline** — click an audio, video, image, or
  file tile to play or view it in the workspace side panel instead of opening
  a separate app.

## App Sidebar

- Added Command Packs: installable deterministic site commands that let agents
  search, read rankings, feeds, and comments, and check login state through
  your real browser session with structured results instead of clicking
  through pages.
- Changed Command Packs to be strictly opt-in: nothing installs
  automatically — each site's pack installs only when you click Install in
  its Websites settings entry.
- Added Command Pack management to the Websites settings page: install a pack
  from a GitHub link, see its commands, and update, disable, or remove it.
- Added a website capability pill to the browser App Sidebar empty state that
  detects the current site and shows its command pack, MCP servers, and
  skills, with a click-through to the site's settings.
- Changed the Websites settings page to a single unified view — no more
  Website/Instruction tabs, URL merged into URL Patterns, and the Command
  Pack shown as a compact row with an icon, name, command count, and switch.
- Added recommended sites (YouTube, Twitter/X, TikTok, Reddit, Hacker News,
  GitHub, and more) as ready-to-use presets pinned to the top of the Websites
  list, each with the site's icon and command count; command packs for 100+
  further sites remain installable from a GitHub link.
- Added API and DOM labels to each command in a pack, with hover explanations,
  so you can tell whether a command uses the site's API or drives the page.
- Changed the Websites list to pin popular sites (Reddit, Hacker News, and
  others) to the top and mark command-pack sites as built-in, so they are not
  accidentally deleted.
- Added automatic command-pack updates on app launch, keeping installed packs
  on their reviewed version.
- Changed the browser automation skill on browser App Sidebar profiles to load
  on demand, keeping browser sidebar prompts lean until page automation is
  actually needed.
- Fixed larger command packs (such as Reddit and Twitter) failing to install
  with a JSON error, by writing the full command list before the installer
  reads it.
- Fixed the App Sidebar empty state jumping while a command pack, MCP server,
  or skill installs; the setup indicator and capability pill now stay in place.
- Changed the Excel, Word, and PowerPoint presets to pin their document-design
  skills (xlsx-design, docx-design, pptx-design) on demand — installed
  automatically when missing — instead of inlining full skill content into
  every sidebar prompt.
- Changed the Office presets' live-editing instructions to verify each
  AppleScript edit by reading the result back from the open document, so
  real-time edits confirm themselves before the agent reports done.
- Changed the Numbers, Pages, and Keynote presets the same way: their
  document-design skills now pin on demand (each app opens the generated
  Excel/Word/PowerPoint file natively), and live AppleScript edits verify
  themselves by reading the result back from the open document.
- Added a Safari preset that pins the safari-mcp server so sidebar agents can
  automate the real Safari browser — navigate, read and fill pages, run
  JavaScript, capture screenshots, and manage tabs with logins intact — with
  setup guidance for Safari's developer settings and macOS permissions.
- Added a Palmier Pro preset that pins its built-in MCP server for AI video
  editing — timeline edits, captions, layouts, color, media generation, and
  export — with setup guidance when the server is not running.
- Added an "I've set this up" button to the App Sidebar setup card that
  re-checks the connection on demand after you finish the manual steps, with a
  clear note when the server is still unreachable, instead of relying on
  dismissing the popover to trigger a refresh.
- Added a Figma preset that pins the Figma desktop MCP server (with setup
  steps for enabling it in Dev Mode) plus Figma's fourteen official skills —
  design-to-code, screen and design-system generation, diagrams, Code Connect,
  motion, FigJam, Slides, SwiftUI, video interaction mapping, and project
  plans — all installed automatically and loaded on demand.
- Fixed the App Sidebar capability and setup popovers overflowing the panel
  when an app pins many capabilities (like Figma's fifteen skills); the list
  now stays within the panel and scrolls without a visible scrollbar, with the
  Customize button pinned at the bottom.
- Fixed installed skills whose global switch is off being offered again as
  recommendations with an Enable button, instead of being recognized as
  already installed.
- Fixed the App Sidebar edge toggle button stealing keyboard focus from the
  app window underneath when clicked; clicks still open the sidebar but the
  active app keeps its focus and text selection highlight.
- Added App Sidebar crash recovery: if Enconvo crashes or is force-killed,
  the sidebars that were open automatically reappear on their app windows the
  next time Enconvo launches.
- Changed the App Sidebar shortcut to open faster by removing an internal
  launch round-trip and pre-warming the web renderer at app launch.
- Fixed the App Sidebar sometimes leaving the app window shrunken after
  closing — the window's original width is now reliably restored even after
  the window was moved or repositioned while the sidebar was open, and even
  when the sidebar closes because the app was hidden or its window closed.
- Fixed clicking the Enconvo Dock icon raising open App Sidebar panels
  detached from their apps; the Dock click now surfaces only the Chat or
  Settings window, opening the Chat window when neither is visible.
- Fixed opening the Settings or Chat window (for example with ⌘, from the
  SmartBar) sometimes bringing a background app's open sidebar to the front
  with it, and the sidebar visibly sinking again when that window closed.
- Fixed a crash when interacting with the model picker after a sidebar's
  conversation state had been released (for example after its app quit or its
  window closed while the panel was kept for reuse).
- Fixed a crash when reading app state for Computer Use on apps that report
  windows or rows with invalid geometry (such as Numbers table rows with
  infinite frames); such elements now appear in the accessibility tree without
  coordinates instead of crashing.
- Fixed Computer Use get_app_state hanging forever on apps with invalid
  accessibility geometry (such as a fresh unsaved Numbers document): element
  data now omits degenerate frames, a native reply that cannot be serialized
  returns an error instead of being silently dropped, and instruction lookups
  time out instead of wedging all future state reads for the same app.

## Providers

- Fixed agent conversations through OpenRouter with Claude models failing with
  "400 Provider returned error" after a tool call, by replaying the
  assistant's intermediate text in its original position in the chat history.

## Commands

- Fixed the TTS (Text To Speech) and Read Aloud commands showing no result
  when run from the SmartBar, by streaming their responses over SSE with a
  live progress indicator while the audio is generated.

## Settings

- Fixed the PopBar instance detail panel showing an empty PopBar Command List
  (only the Add Commands button) by resolving stored preference values on
  first render and ignoring out-of-date command lookups, which also removes
  the controlled-input console error on the PopBar settings page.

## Chat

- Changed clicking an attachment tile in chat messages (audio, video, image,
  or file) to preview it in the workspace side panel with inline playback,
  instead of opening the system default app.
- Added autoplay when an audio or video attachment is opened from a chat
  message, so playback starts as soon as the preview panel appears.
- Changed chat attachment clicks into a toggle, so clicking the same
  attachment again closes the preview panel.
- Changed the message Read Aloud button to play the message's first audio or
  video attachment when the message has no text to read.

## Recordings

- Added transcript editing to the recording/captioning detail page: correct
  any paragraph's text inline or delete a paragraph by clearing it, with
  search and @-mention context automatically re-indexed after edits.
- Fixed the recording rename editor so the editing field keeps the same large
  title font, and added explicit save (checkmark) and cancel buttons while
  renaming.
