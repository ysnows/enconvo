# Enconvo 2.5.5 Beta 7

Thanks for testing Enconvo 2.5.5 Beta 7. This beta is still being prepared.

## Highlights

- **Deleting an AI Command or Agent works again** — the delete button no longer
  fails, asks for confirmation, and fully removes the item along with its chat
  history.
- **Projects gets a management menu** — delete all chats across every project,
  reorder sidebar sections, and sort projects manually, by creation, or by last
  updated.

## Chat Window Polish

- Added a Projects menu with Delete all chats (across every project), sidebar
  section reordering (Projects first or Chats first), and Sort by Manual,
  Created, or Last updated.
- Added drag-to-reorder for projects in the sidebar when Sort by is set to
  Manual order, using a dedicated grab handle so dragging never conflicts with
  clicking a project open.
- Fixed dropdown submenus (Organize sidebar, Sort by) not appearing when the
  parent menu uses a translucent background.
- Fixed a layout gap left behind by a collapsed sidebar section after
  reordering Projects and Chats.
- Fixed the Projects and Chat history "..." menus opening backward over the
  sidebar instead of toward the open space to the right.
- Changed an existing conversation to keep its original agent — the composer's
  agent picker is now read-only mid-conversation instead of looking changeable.
- Added an agent info popover on the conversation composer's agent avatar,
  showing the agent's name and description with a Configure agent button that
  opens it in Settings.
- Added a Workspace Sidebar toggle to the Chat Window header that shows or hides
  the working folder file panel for the open conversation.
- Fixed a chat that belongs to a Project so its workspace is the Project folder —
  the working folder panel now lists the Project's files and messages run in the
  Project directory instead of the agent's default folder.
- Fixed the Workspace Sidebar being slow to open on large projects: the file
  tree now loads folders lazily (top level first, subfolders when expanded) and
  refreshes only open folders instead of re-scanning the whole project, with
  search still covering every folder on demand.
- Changed the Workspace Sidebar to keep its expanded folders when you close and
  reopen it, instead of collapsing and reloading the whole tree.
- Changed the Workspace Sidebar to update from file-system change events instead
  of a timer, so it reflects new or removed files instantly, stays idle when
  nothing changes, and stops watching while its window is hidden (catching up
  when it is shown again).
- Changed the Workspace Sidebar file "+" button to add the file to the current
  window's chat input (Chat Window, Smart Bar, and so on) instead of always the
  Smart Bar.
- Added a Run Mode switch to the Chat Window composer, matching Smart Bar's
  Agent/Chat toggle, with a ⌘I shortcut to cycle between modes.
- Changed native result views the assistant renders in chat to use color:
  metrics now show as color-coded tiles and charts use a themed multi-color
  palette, adapting to light and dark.
- Added a Files group to the Chat Window's "@" mention menu that searches the
  current chat's working folder for matching files and folders as you type, so
  you can reference them without leaving the composer.
- Fixed the "@" Files group not appearing in a brand-new chat that has a
  Project selected but no message sent yet — it now searches that Project's
  folder right away instead of waiting until the first message is sent.
- Fixed the "@" mention menu's folder icons showing an unwanted background
  box, so they now render cleanly like the rest of the icon set.
- Fixed the "@" Files search silently missing real files and folders in
  larger projects, and changed it to skip whatever each folder's own
  .gitignore excludes instead of a fixed list of build folders.
- Changed the sidebar's Projects section, Chats section, and each individual
  project's expanded/collapsed state to be remembered across reloads instead
  of always resetting to expanded.
- Removed the "Loading chats" placeholder that briefly flashed in the sidebar
  while chat history was being fetched.
- Fixed uneven spacing between the attach, agent, and Run Mode buttons in the
  chat composer so the gaps between them now look even.
- Added a hover tooltip to the Run Mode button showing its ⌘I shortcut, so the
  switch-mode shortcut is easier to discover.
- Fixed agents silently losing their tools and agent behavior (replying in
  plain chat mode) after switching models in the new model picker or toggling
  Run Mode — the runtime now understands the new settings format either way.

## Privacy

- Changed anonymous product analytics to report only app start, login/logout,
  and panel display events; command execution, extension install/update/
  uninstall, and settings navigation are no longer tracked.
- Removed all analytics from the extension runtime, including AI provider
  call tracking and the module analytics bridge.
- Added a single anonymous chat analytics event that records the model
  provider, model name, credential type, source window, agent mode, and the
  agent's id and name when a chat message is sent, honoring the same
  analytics opt-out setting.
- Changed analytics identity to a single anonymous per-device ID shared by
  the app, extension runtime, and settings interface; the user's email is no
  longer sent to analytics.
- Added anonymous setup and adoption analytics: permission grant changes,
  successful model provider configuration, extension install/update/removal,
  agent tool usage, and non-LLM capability usage (voice, OCR, image, video,
  web search), all honoring the same analytics opt-out setting.
- Added anonymous voice analytics events counting Dictation sessions (with the
  speech recognition model), Voice Command triggers, and meeting recording
  starts.
- Added anonymous page view analytics for the Settings interface recording
  which settings section is viewed, with all identifiers stripped from the
  reported URL.

## Points And Billing

- Changed the Points settings page to show your balance split into top-up points
  that never expire and subscription points that reset each cycle, each with a
  short explanation of whether it resets.
- Removed the old per-row Balance column from the usage history, replacing it
  with a per-entry breakdown of how much was spent from top-up versus
  subscription points.
- Added a Daily credit usage chart to the Points page that stacks each day's
  spending by model, with a hover breakdown and total.
- Changed the usage history to attribute each AI request to the agent or command
  that actually spent the credits, showing its name instead of "Local Server".
- Changed purchased top-up points and Standard / Premium plan points to never
  expire, while Monthly / Yearly subscription points reset to the plan amount on
  each renewal.
- Fixed credit charges for the new Gemini image models (Nano Banana 2, 2 Lite,
  and Pro) so each image now bills its correct per-image price plus text and
  reference-image tokens, instead of over-charging the fast models and
  under-charging Pro.
- Fixed xAI (Grok) web search on the Enconvo Cloud plan not charging any
  credits, so each search now bills its real cost including tokens and search
  usage.
- Fixed Google Gemini (Enconvo Cloud) audio transcription not charging any
  credits, so it now bills by its token usage.

## Fixes

- Fixed deleting an AI Command in Settings so it removes the command instead of
  showing "Failed to delete AI command", and now also clears the command's chat
  history so no orphaned recent chats are left behind.
- Fixed deleting an Agent in Settings, which was failing the same way, so it now
  removes the agent and its chat history.
- Fixed the Settings footer account shortcut so it scrolls the Account menu
  item into view after opening the account page.
- Changed the AI Command and Agent delete confirmation to state that the item's
  chat history (conversations and messages) will also be permanently removed.
- Changed AI Command instructions to be sent as part of the system prompt
  instead of being attached to the user's message, so models treat them with
  proper instruction priority.
- Changed agent conversations on Anthropic models to keep their prompt cache
  warm for up to an hour, so follow-up messages after a pause respond faster
  and cost less instead of re-processing the whole conversation.
- Changed the agent prompt layout to keep session-stable instructions in the
  cached prompt prefix, raising measured follow-up cache hit rates from about
  87% to about 99%.
- Changed the agent's dynamic context to fetch the live context list (open
  windows, tabs, and apps) on demand through tools instead of embedding it
  into every message, making conversations faster and cheaper while keeping
  current window awareness available when a task needs it.
- Fixed the agent duplicating its abilities instructions on every tool call
  within a turn, which inflated token usage and broke prompt caching for
  tool-using conversations.
- Changed MCP tools in the agent's tool list to show only a name and short
  summary, with full parameter schemas fetched on demand before first use —
  cutting agent prompt size by about 40% when large MCP servers such as
  Notion are connected.
- Fixed MCP tool schema lookups failing after a server's idle connection
  closed — the server now reconnects automatically when its schema is
  requested.
- Changed the agent's built-in capability descriptions (web search,
  transcribe) and tool instructions to list only required parameters, with
  optional ones fetched on demand — trimming another ~15% off every agent
  request.
- Changed the agent prompt so session-specific details (session id,
  workspace, model, current time) travel with each message instead of the
  system prompt, making the system prompt identical across conversations — a
  brand-new conversation's first message now reuses the prompt cache warmed
  by earlier ones (measured first-message hits went from ~0% to ~98%), and
  switching models mid-conversation no longer invalidates the cache.
- Fixed command settings (such as Translate) losing their configured AI model
  when the global default provider was set to a different provider — the saved
  provider and model are now shown and used correctly instead of silently
  following the global default.
- Fixed the provider dropdown in command settings flickering back to the
  previous provider right after switching to a new one — the panel now keeps
  showing the provider you just selected.
- Removed the internal caiyun, deep_translate, deepl, and youdao API endpoints
  from the Translate plugin, so its command list shows only Translator and
  Screenshot Translate.
- Fixed the Smart Bar model picker opening on the wrong provider when several
  providers list a model with the same name — it now locates the provider that
  is actually configured for the command.
- Fixed the model picker offering providers that displayed another provider's
  model list (Arli AI, Azure OpenAI, Perplexity, SiliconFlow, LM Studio,
  Together AI), which could bind a model those providers don't actually host.
- Fixed the Smart Bar running a previously used bot (and firing its paste
  post-action) instead of the selected default agent when sending from an empty
  conversation, including the case where re-selecting the default agent in the
  picker didn't take effect.
- Added the attach button (screenshot, live camera, and file upload) to the
  Dynamic Island conversation view's actions bar, matching the input view.
- Fixed the Dynamic Island top toolbar shifting vertically when switching
  between the input view and the conversation view — it now stays at a fixed
  position across modes.
- Fixed input method candidate suggestions being hidden behind the Dynamic
  Island when typing a follow-up in the conversation view — candidates now
  stay visible while composing, matching the input view.
- Changed the MCP plugin to be visible in the Settings Plugins page and in the
  "@" plugin search, so MCP server management can be opened like any other
  plugin.
- Fixed private extension API endpoints marked "@private" still appearing as
  commands on a plugin's detail page, so internal routes (such as MCP's install
  and startup hooks) are no longer listed alongside real commands.
- Added an MCP plugin skill so the assistant knows how to register, connect,
  and call tools on MCP servers on your behalf.
- Changed OAuth completion pages for credentials providers to use Enconvo's
  dark confirmation layout with the Enconvo logo.
- Fixed AI image generation ignoring your selected image model and silently
  using an older Gemini model when the assistant didn't name one explicitly, so
  images now honor the model you chose (such as Nano Banana 2 Lite).
- Added an in-app plan page to compare and buy Standard, Premium, and Cloud
  plans, plus a Top Up option for buying extra point packs, without leaving the
  app.
- Changed plan purchase, top-up, and Manage Subscription to open Stripe's
  checkout and billing pages directly instead of routing through the
  enconvo.com website.
- Fixed the Points balance numbers briefly compressing the row height on first
  load, so the balance no longer jumps while the count-up animation runs.
- Fixed the app UI freezing and stuttering while AI responses stream in —
  incoming extension messages are now decoded off the main thread instead of
  blocking the interface.
- Fixed several seconds of high CPU usage and UI lag after extensions sync
  (such as right after launch) by making bulk hotkey registration and command
  loading dramatically cheaper, including caching installed command files so
  they are no longer re-read and re-parsed on every lookup.
- Fixed window tracking repeatedly querying every running app to locate
  Finder, which slowed down capturing open windows for context awareness.
- Removed the daily free-usage run limit for non-subscribers — commands no
  longer stop with an "exceeded the free usage limit" upgrade prompt, and run
  counts are no longer tracked.
- Removed the legacy Premium/Account page from the classic settings window;
  plans and billing are managed from the new in-app Plans page instead.
- Fixed agent mode silently stopping after "Thought for … seconds" with
  DeepSeek (and other models) when the model called a tool name it wasn't
  offered — the agent now tells the model the tool doesn't exist and lists the
  available tools so it can correct itself and finish the task.
- Fixed DeepSeek thinking-mode requests through OpenAI-compatible endpoints
  failing with "400 The reasoning_content in the thinking mode must be passed
  back to the API" by replaying each tool call's reasoning content in
  follow-up requests, as the DeepSeek API requires.
- Changed the first-run onboarding guide into a transparent frosted-glass window
  that matches the app's other panels, with a draggable top, a cleaner provider
  setup screen whose model list scrolls on its own, and the Continue button
  anchored in the bottom-right corner.
- Removed the Companion Orb step from the onboarding guide.
- Fixed chats failing with "400 unknown variant disabled" on OpenAI-compatible
  providers (such as NVIDIA NIM) when a model's reasoning was set to Disabled —
  the app now omits the reasoning parameter instead of sending an unsupported
  "disabled" value.
- Fixed LM Studio missing from the Smart Bar and Chat Window model pickers — it
  now lists your actual LM Studio models from its local API (with context
  length and vision detection) instead of borrowing OpenAI's model list,
  refreshes automatically when the picker is opened, and keeps the last-seen
  models available while the LM Studio server is off.
- Added a Sign-in step to the onboarding guide with a Log in button that returns
  you to the guide after signing in, instead of jumping to the Settings window.
- Fixed the Chat Window model picker's provider list scrolling on its own when
  clicking a provider — it now only scrolls to the current provider when the
  picker first opens.
- Changed the model picker to one consistent highlight language: the selected
  provider, model, and reasoning level are marked with a checkmark, while
  hovering shows a lighter background tint.
- Changed multi-turn agent conversations to replay each turn's context
  byte-identically from history, so follow-up messages reuse the prompt cache
  for the entire previous conversation — including tool calls and results —
  instead of re-processing it every turn (measured: a follow-up message now
  re-sends only ~100-500 fresh tokens instead of the whole prior turn).
- Changed the agent's memory context to include only usage rules and API
  schemas — per-turn memory preloading was removed, so requests start faster
  and the prompt no longer changes when stored memories change; the agent now
  searches memory on demand instead.
