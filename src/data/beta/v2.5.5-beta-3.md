# Enconvo 2.5.5 Beta 3

This beta completely rebuilds MCP server management from the ground up. MCP
runs more reliably, is easier to configure, and works seamlessly with the
agent's tool system.

## Highlights

- **MCP server management redesigned** — a new dedicated settings pane replaces
  the old extension-based approach with a simpler, more stable architecture.
- **Agent mode starts faster** by initializing agent context in parallel.
- **Video transcription is faster** because Enconvo now extracts audio from
  video files automatically before transcribing.
- **Settings are cleaner** with Extensions renamed to Plugins and a new MCP
  Servers pane.

## MCP Servers

- Redesigned MCP server management with a new config-driven architecture that
  replaces the old per-extension model, making servers easier to add, configure,
  and maintain.
- Added a dedicated MCP Servers settings pane with server list, status
  indicators, tool inspection, and import from JSON config.
- Added OAuth authorization flow for MCP servers that require sign-in.
- Added support for both stdio and HTTP (Streamable HTTP) transports with
  connection pooling and automatic idle disconnect.
- Added dynamic MCP tool schema lookup so the agent can inspect tool parameters
  for any connected MCP server.
- Added automatic migration of legacy MCP extensions to the new config format
  on app startup — existing servers are preserved.
- Fixed MCP tool calls passing framework metadata as tool arguments to MCP
  servers.

## Agent And Media Workflows

- Improved agent mode startup by preparing agent context in parallel.
- Added automatic audio extraction from video files in transcription workflows.

## SmartBar & SideBar 

- Added default agent configuration for SmartBar and AppSideBar.
- Improved App Sidebar dismiss behavior.
- Improved global Voice Command trigger logic.

## Settings
- Opened external WebView links in the default browser instead of trapping them
  inside Enconvo.
- Removed older unused chat/plugin views and legacy command surfaces that are no
  longer part of the current app experience.
