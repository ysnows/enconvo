# Enconvo 2.5.4 Beta 2

- [100%] Cleaner chat answer formatting
  - Chat With AI now avoids fenced code blocks unless the user asks for code, commands, structured data, logs, or exact copyable text.
  - Formulas should stay in prose or normal Markdown math instead of code-style boxes.

- [100%] Correct OpenRouter Claude reasoning effort options
  - OpenRouter Claude Opus and Sonnet model ids now match effort profiles by model family and version, even when naming order or separators vary.
  - Opus 4.7 and 4.8 show adaptive Low, Medium, High, X-High, and Max effort choices instead of legacy token-budget choices.
  - Other Claude fallback models, including Sonnet 4.5 and latest aliases, now use the non-numeric Low, Medium, High, and Max effort list.

- [100%] More reliable Dynamic Island hover activation
  - Fixed repeated compact Dynamic Island hover entries sometimes failing to open.
  - Compact hover validation no longer cancels activation when the pointer is inside the padded hover target but outside the visible pill.
  - Background agent stream preview no longer cancels pending hover activation unless the Dynamic Island mode actually changes.

- [80%] Chat message list scroll performance
  - Moved Streamdown props (linkSafety, components, plugins, rehypePlugins) to module-level constants so React.memo comparators see stable references and skip unnecessary re-renders.
  - Added CSS content-visibility: auto on cached message containers so the browser skips layout/paint for off-screen messages.
  - Extracted per-session values from shared Map state before useMemo to avoid recalculating when other sessions change.
  - Source-level regression tests verify the stable-reference and containment patterns.
