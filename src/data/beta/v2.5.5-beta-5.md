# Enconvo 2.5.5 Beta 5

Thanks for testing Enconvo 2.5.5 Beta 5. This build focuses on workflows,
better agent output, faster model responses, richer app context, and a smoother
Dynamic Island experience.

## Highlights

- **Voice Command is more reliable on macOS 27**, fixing issues that could
  prevent voice commands from running correctly.
- **LLM prompt caching is much more effective**, improving cache hit rates so
  repeated model work can respond faster and use fewer tokens.
- **Workflows are now available in Enconvo**, with support for using Enconvo
  features, MCP tools, and VS Code tools inside workflow steps.
- **AI Agent can help create workflows**, so you can describe the automation you
  want and let the agent draft or edit the workflow for you.
- **Skills and Plugins have new management pages**, making installed tools,
  plugin details, API endpoints, and settings easier to browse and maintain.
- **Generated UI results can now appear directly in Enconvo**, allowing agents,
  tools, and workflows to show richer cards, tables, charts, and interactive
  results instead of plain text only.
- **Task completion can open Dynamic Island and send system notifications**,
  with settings for automatically showing the result and notifying you when an
  agent session finishes.
- **Quick Memory makes saving context faster**, with double-tap right Shift to
  save the current app context to your knowledge base and long-press right Shift
  to save a voice note.
- **Mentions now support workflows and MCP servers**, so `@Workflows` and
  `@MCPs` can be added as chat context or used by agents.
- **App-specific mentions can add real content**, including files from Finder
  App Sidebar, notes from Notes, and email from Mail.app.
- **Dynamic Island supports richer input**, including adding screenshots,
  choosing files, and toggling automatic read-aloud for responses.
- Fixed ten plugin API crashes on missing parameters — config/set, workflow/get, web_fetch, knowledge_base add/update/attachment content, MLX VLM chat/generate, plugin detail, and the Enconvo Cloud LLM entry now return clear errors instead of server crashes.
- Fixed adding text to a knowledge base without specifying one crashing on a database constraint; it now falls back to the default knowledge base.
- Fixed the OCR command crashing when called without an image; it now reports that image_files is required.
- Changed workflow templates to render objects and arrays as JSON automatically, eliminating silent "[object Object]" output when interpolating step results.
