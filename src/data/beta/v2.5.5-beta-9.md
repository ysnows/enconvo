# Enconvo 2.5.5 Beta 9

Thanks for testing Enconvo 2.5.5 Beta 9. This build focuses on fully on-device
dictation: new CoreML models run on the Apple Neural Engine and can be managed
from the same Dictation Models page as the existing local models.

## Highlights

- **New CoreML dictation models** bring Parakeet, SenseVoice, and Paraformer to
  fully on-device transcription on the Apple Neural Engine.
- **CoreML model management** is now built into Dictation Models, with download
  progress, cancel, load, unload, and delete controls.
- **Faster local dictation startup** avoids unnecessary network checks for
  models that are already downloaded.

## On-Device Transcription

- Added CoreML dictation models powered by FluidAudio: Parakeet (multilingual,
  English, and Japanese), SenseVoice (Chinese, Cantonese, English, Japanese,
  Korean), and Paraformer (Mandarin Chinese).
- Added download, progress, cancel, load, and delete controls for CoreML
  transcription models in the Dictation Models settings.
- Changed the bundled on-device audio engine to the latest FluidAudio release for
  faster transcription and real download progress reporting.
- Fixed the on-device model size labels to show the actual download size instead
  of the full repository size, so models read as their true few-hundred-MB size.
- Added the specific language name to single-language on-device models (Japanese,
  Chinese, English) and a hover tooltip listing every supported language.
- Fixed on-device transcription model status so it no longer stays stuck on
  "downloading" after a download has finished.
- Changed on-device transcription to keep only one model loaded at a time to
  reduce memory use.
- Changed on-device model downloads to show live "downloaded / total MB
  (percentage)" progress, and to keep showing progress after you leave and
  return to the Dictation Models page.
- Changed CoreML transcription model downloads to use accelerated XET transfers
  that fetch only each model's required files, so they download several times
  faster and resume instantly.
- Added an Unload button to loaded on-device transcription model cards (hover
  the Loaded badge) to free the model from memory while keeping it on disk.
- Fixed a deleted on-device model that was in use ("selected") showing no way to
  re-download it — its card now shows the Download button again.
- Changed the Dictation Models list to surface recommended models first (Soniox
  Realtime, Parakeet TDT V3 CoreML, and Qwen ASR).
- Changed each dictation model card to reveal its Delete and Settings actions on
  hover, keeping the list uncluttered.
- Fixed the delete and download confirmation dialogs opening without their
  fade-and-zoom animation.
- Changed on-device dictation models to load faster by skipping a redundant
  HuggingFace network check when the model is already downloaded.
- Removed App Context from the on-device Qwen ASR models, which ignore it; their
  Dictionary (hot words) still applies.

## Startup Performance

- Changed the extension-sync message sent to the app after startup to carry
  only hotkey data instead of every command's full configuration, removing
  seconds of background parsing on every sync — the top hotspot in user CPU
  traces.
- Changed the installed-applications scan to be skipped entirely when no app
  has been installed, removed, or updated since the last scan, eliminating
  repeated per-app disk and LaunchServices lookups at launch.
- Added a startup log line recording how many extensions were reinstalled per
  sync, so machines stuck re-installing everything on every launch can be
  identified from their logs.

## Chat

- Changed the per-turn environment context sent with each message to be
  prompt-cache friendly, so long conversations reuse cached context across
  turns instead of re-processing the whole transcript, responding faster and
  using fewer tokens.

## Fixes

- Fixed the "Create New AI Command" button being cut off below the visible
  area at the bottom of the AI Commands list, so it is always reachable.
- Fixed the "Add New Channel" button being cut off at the bottom of the IM
  Channels list, so it is always reachable.
